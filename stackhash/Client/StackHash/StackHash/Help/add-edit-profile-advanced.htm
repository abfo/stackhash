WinQual API

These settings allow you to fine tune how StackHash works with the WinQual API.

Timeout sets the number of minutes to wait for a response from WinQual.

Cab download failure limit controls how many failed downloads can occur before a synchronization attempt will be abandoned.

Per-request retries is the number of times StackHash will retry a failed request before a synchronization attempt will be abandoned.

Syncs before automatic resync triggers a full resynchronization with WinQual after the specified number of normal synchronizations. During a normal sync StackHash adds events, hits and cabs that have been added to WinQual since the last sync completed. Resynchronization processes all events, hits and cabs and can pick up data that may have been missed during the last sync. Setting this value to 1 will force StackHash to always resynchronize.

Check Retry after failed synchronization to schedule another attempt after the number of minutes specified in Delay before retry below.

Check Enable sync for new product mappings to automatically start synchronizing new products. If unchecked you must manually right-click each new product and choose Enable Synchronization.

Checking Force update of all script results will cause each script that is set to run automatically to run again on each cab that has been downloaded. This setting will affect the next sync only and will be reset once all scripts have been run again. The default is to only run scripts if a result for a cab is not present and the script has not changed since the result was added.
