Main Window - Mapped Products

The mapped products view lists products mapped to your WinQual account for the current profile. This is the default view when you run StackHash. You can return to this view at any time from View -> Home.

When you select a product the charts on the right will refresh to help you visualize the hits associated with all events for that product. The charts are interactive — hover to see a tooltip with specific data, double-click a chart element to search for events that match the highlighted data.

The world map shows an approximate distribution of hits for the selected product. This chart is based on the locale reported with each hit. Some countries use multiple locales and some countries do not have an associated locale.

Double click a product or click the List Events link to show the Event List view.

A context menu is available when you right-click a mapped product. From this menu you can enable or disable a product for synchronization, copy the StackHash URL, export information from the product to XML, set product properties and open the product page on the WinQual site.

If no products are shown on this view check the following:

  1. StackHash allows you to hide products that are disabled for synchronization. You can toggle the visibility of these products from the View menu.
  2. Refresh the current view: hit Ctrl-R, or View -> Refresh.
  3. Check that you are connected to the StackHash service and that an active profile is available in Tools -> Client Options -> Service Connection.

Mapped Products

Synchronize Yes if the product is enabled for synchronization, No if not. Disabled products are grayed out in the list and do not count towards the event limit for your license Right-click the product and choose Enable Synchronization or Disable Synchronization to change this setting.
ID The WinQual Id for the product mapping.
Name Name of the product (as set in the Microsoft Product Feedback Mapping Tool when the mapping was created).
Version Version of the product (as set in the Microsoft Product Feedback Mapping Tool when the mapping was created).
Events on WinQual Number of events (buckets) currently available for the product mapping on WinQual. If this number is higher than Events in StackHash after synchronization please see Missing Events? on the StackHash Blog.
Events in StackHash Number of events (buckets) currently available for the product in StackHash. WinQual stores data for the past 90 days so over time this number is likely to be higher than Events on WinQual.
Responses Number of responses registered on WinQual for the product.
Created Local date and time that the product mapping was created on WinQual.
Modified Local date and time that the product mapping was last modified on WinQual.
Last Sync Started Local date and time that the last synchronization was started for the product. If blank StackHash has never synchronized the product with WinQual.
Last Sync Completed Local date and time that the last synchronization was completed for the product. If blank StackHash has never completed synchronization of the product with WinQual. If earlier than Last Sync Started then synchronization is either still in progress or has failed (see Tools -> Synchronization Report for more information).
Cab Collection Policy The cab collection policy for this product. Note that the collection policy controls how many of the available cabs are downloaded from WinQual. To request more cabs you must visit the WinQual web site. Default Collection Policies can be set for the profile in Service Options. To override the default policies for the current product right-click it and choose Properties.
Event Collection Policy The event collection policy for this product. Determines the hit threshold before cabs are downloaded for events. Default Collection Policies can be set for the profile in Service Options. To override the default policies for the current product right-click it and choose Properties.
Event Filter The current event filter. If a filter is enabled events below a hit threshold are not displayed in the client. The default Display Filter can be set in Client Options. To override the filter for the current product right-click it and choose Properties.

Note that columns can be re-ordered in the client and so might not be in the order listed above. You can re-order columns by clicking and dragging the header.
