Service Status

The top list shows all profiles configured on the service, including if the profile is active or not and if the most recent attempt to logon to WinQual failed or succeeded.

Click a profile for detailed statistics in the tasks list. This shows:

Type:The service runs various tasks for each profile, including WinQual synchronization, crash dump analysis, purge and timer tasks responsible for scheduling the other tasks.
State:Timer tasks should always be in the Running state. Other tasks should be Running or NotRunning.
Last Exception:Reports details of the last error for each task. If empty then the task has not reported an error. Pay attention to the Last Failed date as the Last Exception may not indicate a current problem with the task.
Last Duration:Time taken to run the task in seconds. This is not meaningful for the timer tasks.
Run Count:Total number of times that the task has been launched.
Success Count:Total number of times that the task has completed successfully.
Failed Count:Total number of times that the task has failed.
Last Started:Date and time that the task was last started (UTC - Coordinated Universal Time).
Last Succeeded:Date and time that the task last completed successfully (UTC - Coordinated Universal Time).
Last Failed:Date and time that the task last failed (UTC - Coordinated Universal Time).

Tasks started based on a request from the client (move index, copy index, download cab and bug report) can be aborted if running. Right click the task and choose Abort Task.

Click Refresh to update the Service Status window.

Click Plugin Diagnostics to view troubleshooting information reported by any installed plugins. Select a profile first to view profile specific plugin diagnostics.
